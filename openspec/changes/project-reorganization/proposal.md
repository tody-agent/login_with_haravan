## Why we're doing this
### Qualified Problem
**For:** Đội ngũ phát triển và AI Agents (CodyMaster).
**Who:** Đang gặp khó khăn trong việc phân biệt giữa source code Frappe, tài liệu public (VitePress), tài liệu nội bộ của Agent, và các file cấu hình.
**The:** Cấu trúc repository hiện tại
**That:** Đang trộn lẫn tất cả mọi thứ ở thư mục gốc (root) và thư mục `docs/`.
**Unlike:** Một cấu trúc chuẩn hóa, nơi source code, public docs, internal plans và security files được phân tách rõ ràng.
**Our approach:** Quy hoạch lại các thư mục theo chức năng, đảm bảo tuân thủ yêu cầu deploy của Frappe Cloud, tách biệt tài liệu public và nội bộ.

## What's changing
### Options Evaluated

#### Option A: Monorepo Approach (Strict Separation)
- **Approach:** Đưa app vào `apps/login_with_haravan`, docs vào `apps/docs`.
- **Pros:** Cực kỳ gọn gàng ở root.
- **Cons:** **KHÔNG THỂ THỰC HIỆN ĐƯỢC**. Frappe Cloud và lệnh `bench get-app` yêu cầu `setup.py` và thư mục app (`login_with_haravan`) phải nằm ngay ở root của repository.
- **Effort:** Lớn (phá vỡ CI/CD và deployment hiện tại).

#### Option B: Frappe-Native + Segmented Docs (Recommended)
- **Approach:** Giữ Frappe App ở root. Dọn dẹp và phân loại rạch ròi các file tài liệu và cấu hình.
- **What changes:**
  1. **Frappe App (Giữ nguyên ở root):** `login_with_haravan/`, `setup.py`, `pyproject.toml`, `requirements.txt`.
  2. **Public Docs (`docs/`):** Chỉ chứa tài liệu sẽ publish qua VitePress (Architecture, API, Database, Deployment). KHÔNG chứa tài liệu nội bộ.
  3. **Internal / Ideas (`openspec/` & `internal/`):** Chứa các kế hoạch, SOP của Agent (`NEXT_STEPS_FOR_AGENT.md`, `SITE_CONFIG_HANDOFF.md`, các folder JTBD, Personas).
  4. **Scripts (`scripts/` hoặc root):** `ship.sh`, `test_gate.sh` (Nên để root để dễ gọi).
  5. **Security & Ignored files:** Gom các file cấu hình nhạy cảm và build cache vào `.gitignore` / `.npmignore`.
- **Effort:** S (Nhỏ - khoảng 15-30 phút di chuyển file và cập nhật path).
- **Pros:** Tuân thủ 100% chuẩn Frappe Cloud; VitePress không bị lộ tài liệu nội bộ của Developer/Agent.
- **Cons:** Root vẫn có một vài file cấu hình cần thiết.

### Recommendation
**Chọn Option B: Frappe-Native + Segmented Docs**

### Chi tiết tổ chức (Quy hoạch thư mục)

#### 1. Thư mục Source Code (Deploy lên Frappe Cloud)
- `login_with_haravan/`: Source code chính.
- `setup.py`, `pyproject.toml`, `requirements.txt`, `MANIFEST.in`: Cấu hình Python package.
- `package.json`, `package-lock.json`: Tạm thời để ở root nếu dùng chung, hoặc chuyển vào `docs/` nếu chỉ phục vụ VitePress.

#### 2. Thư mục Public Docs (VitePress - Deploy lên Cloudflare Pages)
Chỉ để những gì End-User hoặc External Developer cần đọc.
- `docs/api/`
- `docs/architecture.md`
- `docs/data-flow.md`
- `docs/database.md`
- `docs/deployment.md`
- `docs/index.md`

#### 3. Thư mục Kế hoạch & Ý tưởng (Ideas / Internal / Agent)
KHÔNG deploy, chỉ dùng cho nội bộ và AI Agents.
- `openspec/`: Nơi chứa Fission-AI OpenSpec, proposals, kế hoạch phát triển mới.
- Khuyên dùng thêm thư mục `.agent/` (hoặc gom vào `openspec/`) để chứa:
  - `docs/NEXT_STEPS_FOR_AGENT.md` -> Chuyển thành `openspec/AGENT_NEXT_STEPS.md`
  - `docs/AGENT_HANDOFF.md` -> Chuyển thành `openspec/AGENT_HANDOFF.md`
  - `docs/SITE_CONFIG_HANDOFF.md` -> Chuyển thành `openspec/SITE_CONFIG.md`
  - Cấu hình public còn lại nằm ở `docs/guide/configuration.md`
  - Troubleshooting public còn lại nằm ở `docs/guide/troubleshooting.md`
  - Các thư mục `docs/jtbd/`, `docs/personas/`, `docs/sop/` hiện đang nằm trong `docs/` nên chuyển qua đây.
- `AGENTS.md`: Giữ ở root như là file Manifest định tuyến cho Agent.

#### 4. Danh sách các file CẦN IGNORE (Bảo mật & Tránh deploy rác)
Đảm bảo `.gitignore` phải chứa:
- Môi trường & Secret: `.env`, `site_config.json`, `*.pem`, `*.key`
- Build cache Frappe: `__pycache__/`, `*.pyc`, `*.egg-info/`, `build/`, `dist/`
- Frontend/Docs cache: `node_modules/`, `docs/.vitepress/cache/`, `docs/.vitepress/dist/`, `.wrangler/`
- File hệ thống: `.DS_Store`, `*.log`
