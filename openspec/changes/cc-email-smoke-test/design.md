# Design: CC Email Smoke Test
Generated by: cm-planning
Date: 2026-05-01

## Goal

Verify that an agent-created Helpdesk ticket on `https://haravan.help/helpdesk/tickets/new`
can persist `HD Ticket.custom_cc_emails` and create an outbound email queue item for
the CC recipient.

## Test Data

| Field | Value |
| --- | --- |
| Agent account | `maryfranceslatecia8151@gmail.com` |
| Raised by | `sociads@gmail.com` |
| CC Emails | `hai.minh@outlook.com` |
| Site | `haravan` / `haravandesk.s.frappe.cloud` |

Do not write the supplied password into logs or repo artifacts.

## Plan

1. Preflight production state through MCP.
   - Confirm installed `login_with_haravan` version.
   - Confirm `custom_cc_emails` field exists.
   - Confirm whether the temporary `HD Ticket - CC Created Notification` Server Script is enabled.
   - Capture baseline `Email Queue` and `Error Log` timestamps.

2. Browser smoke test through the public Helpdesk UI.
   - Open `https://haravan.help/helpdesk/tickets/new`.
   - Log in with the supplied agent account if needed.
   - Create a ticket with `raised_by = sociads@gmail.com` and `custom_cc_emails = hai.minh@outlook.com`.
   - Use a unique subject containing the test timestamp.
   - Save screenshots and a run log under `/tmp/haravan-cc-smoke-*`.

3. Backend verification.
   - Read the created `HD Ticket`.
   - Verify `custom_cc_emails` is normalized and persisted.
   - Verify an `Email Queue` record exists for `reference_doctype = HD Ticket`,
     `reference_name = <ticket id>`, and recipient `hai.minh@outlook.com`.
   - Check recent `Error Log` records for ticket id, CC notification script, or email queue failures.

4. Debug and fix if any acceptance criterion fails.
   - If no queue exists, inspect live app version and temporary Server Script behavior.
   - If queue exists but is errored, inspect the queue record and error log.
   - If duplicate queues exist, keep the production workaround only until app `0.1.6+` is deployed,
     then disable the temporary Server Script.

## Acceptance Criteria

- Ticket is created successfully from the public Helpdesk UI.
- `HD Ticket.raised_by` is `sociads@gmail.com` or the Helpdesk-created requester equivalent is clearly recorded.
- `HD Ticket.custom_cc_emails` equals `hai.minh@outlook.com`.
- At least one `Email Queue` or sent `Communication` path targets `hai.minh@outlook.com` for the created ticket.
- No new related `Error Log` is created during the smoke test.

## Notes

The previous failure mode was silent: `custom_cc_emails` persisted but no `Email Queue`
was created. This test therefore treats UI success alone as insufficient evidence.
