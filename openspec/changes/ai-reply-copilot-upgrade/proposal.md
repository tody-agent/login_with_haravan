# Proposal: AI Reply Copilot Upgrade
Generated by: cm-brainstorm-idea
Date: 2026-04-30

## Why we're doing this

### Qualified Problem

**For:** Haravan Helpdesk agents handling merchant tickets.
**Who:** Need fast, accurate customer-facing replies based on the ticket content and latest exchanges.
**The:** current AI reply suggestion is a basic draft dialog.
**That:** should understand the ticket need, show likely scenarios, and provide ready-to-edit Vietnamese reply drafts.
**Unlike:** the current state where AI may return one weak option, limited context, and sometimes unaccented Vietnamese UI/prompt text.
**Our approach:** keep the current Desk-managed endpoints stable, but upgrade the prompt contract, context payload, option normalization, and dialog UI.

### 9 Windows Analysis

| Level | Past | Present | Future |
| --- | --- | --- | --- |
| Super-system | Agents manually read ticket history and write replies. | Helpdesk has AI scripts but quality depends on sparse prompt data. | AI should act as a bounded copilot, not an autonomous sender. |
| System | Reply suggestion was a single textarea. | API can return `options`, but UI does not let agents compare cases. | API returns three scenario-aware drafts in one call, with auto-selected best option. |
| Sub-system | Prompt collected subject, description, and a few messages. | Recent conversation order/context is shallow, Vietnamese text is partly unaccented. | Prompt includes ticket fields, recent exchanges, internal context, confidence, missing context, and accented Vietnamese requirements. |

### Root Causes

1. Technical: AI response schema is too loose and not normalized before returning to the UI.
2. Product: The dialog assumes one reply path even when the ticket has multiple plausible customer intents.
3. Design: UI lacks radio choices, confidence/missing-context indicators, and Vietnamese accents.

### Impact if not addressed

- Agents spend time rewriting poor drafts.
- Ambiguous tickets may get overconfident or mismatched replies.
- Unaccented Vietnamese reduces trust in the feature.
